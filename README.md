# Исследование эффективности моделей семантического поиска

Этот проект посвящен сравнительному анализу трех моделей (MiniLM, MPNet, E5) для задач семантического поиска по программному коду. Работа выполнена в рамках учебной практики БВТ25.

## Структура проекта
- `dataset_v1.0/` — папка с исходными данными (датасет фрагментов кода Python/Java и тестовые запросы).
- `semantic_search_analysis.ipynb` — Jupyter Notebook с реализацией алгоритмов, расчетом метрик и визуализацией.
- `requirements.txt` — список ключевых зависимостей для запуска проекта.

## Основные этапы работы
1. **Подготовка данных:** Загрузка и индексация фрагментов кода с использованием различных embedding-моделей[cite: 1, 2].
2. **Семантический поиск:** Реализация поиска по векторному сходству[cite: 1, 3].
3. **Метрики качества:** Оценка эффективности моделей через Precision@3 и MRR.
4. **Визуализация:** Использование t-SNE для построения карты распределения эмбеддингов.
5. **Анализ ошибок:** Выявление причин снижения точности и анализ языковой адаптации моделей.

## Визуализация результатов
В ноутбуке реализована визуализация векторных представлений (эмбеддингов) с помощью алгоритма **t-SNE**.
- Каждая точка на графике соответствует фрагменту кода[cite: 1].
- Цвета точек соответствуют тематическим категориям, определенным в `categories.json`[cite: 1].
- Анализ графиков позволяет наглядно оценить, насколько хорошо модель "понимает" семантическую близость функций[cite: 1].
- Также приведена визуализация типичных кейсов, где выбранная модель (E5-small) показывает наиболее высокую точность[cite: 1].

## Результаты исследования
В ходе исследования было установлено, что модель **E5-small** является наиболее эффективной для данной задачи, обеспечивая точность Precision@3 на уровне **96%**[cite: 1]. Она значительно превосходит MiniLM (68%) и MPNet (84%) за счет специализированного обучения на поисковых задачах и использования префиксов для разделения запросов и документов[cite: 1].
